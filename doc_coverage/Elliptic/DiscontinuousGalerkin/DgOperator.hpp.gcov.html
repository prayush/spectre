      40             : /*!
      41             :  * \brief Functionality related to discontinuous Galerkin discretizations of
      42             :  * elliptic equations
      43             :  *
      44             :  * The following is a brief overview of the elliptic DG schemes that are
      45             :  * implemented here. A publication that describes the schemes in detail is in
      46             :  * preparation and will be referenced here.
      47             :  *
      48             :  * The DG schemes apply to any elliptic PDE that can be formulated in
      49             :  * first-order flux-form
      50             :  *
      51             :  * \f{equation}
      52             :  * -\partial_i F^i_\alpha + S_\alpha = f_\alpha(x)
      53             :  * \f}
      54             :  *
      55             :  * in terms of fluxes \f$F^i_\alpha\f$, sources \f$S_\alpha\f$ and fixed-sources
      56             :  * \f$f_\alpha(x)\f$. The fluxes and sources are functionals of the "primal"
      57             :  * system variables \f$u_A(x)\f$ and their corresponding "auxiliary" variables
      58             :  * \f$v_A(x)\f$. The uppercase index enumerates the variables such that
      59             :  * \f$v_A\f$ is the auxiliary variable corresponding to \f$u_A\f$. Greek indices
      60             :  * enumerate _all_ variables. In documentation related to particular elliptic
      61             :  * systems we generally use the canonical system-specific symbols for the fields
      62             :  * in place of these indices. See the `Poisson::FirstOrderSystem` and the
      63             :  * `Elasticity::FirstOrderSystem` for examples.
      64             :  *
      65             :  * The DG discretization of equations in this first-order form amounts to
      66             :  * projecting the equations on the set of basis functions that we also use to
      67             :  * represent the fields on the computational grid. The currently implemented DG
      68             :  * operator uses Lagrange interpolating polynomials w.r.t.
      69             :  * Legendre-Gauss-Lobatto collocation points as basis functions. Support for
      70             :  * Legendre-Gauss collocation points can be added if needed. Skipping all
      71             :  * further details here, the discretization results in a linear equation
      72             :  * \f$A(u)=b\f$ over all grid points and primal variables. Solving the elliptic
      73             :  * equations amounts to numerically inverting the DG operator \f$A\f$, typically
      74             :  * without ever constructing the full matrix but by employing an iterative
      75             :  * linear solver that repeatedly applies the DG operator to "test data". Note
      76             :  * that the DG operator applies directly to the primal variables. Auxiliary
      77             :  * variables are only computed temporarily and don't inflate the size of the
      78             :  * operator. This means the DG operator essentially computes second derivatives
      79             :  * of the primal variables, modified by the fluxes and sources of the system
      80             :  * as well as by DG boundary corrections that couple grid points across element
      81             :  * boundaries.
      82             :  *
      83             :  * \par Boundary corrections:
      84             :  * In this implementation we employ the "internal penalty" DG scheme that
      85             :  * couples grid points across nearest-neighbor elements through the fluxes:
      86             :  *
      87             :  * \f{align}
      88             :  * (n_i F^i_v)^* &= \frac{1}{2} n_i \left(
      89             :  * F^i_v(u^\mathrm{int}) +
      90             :  * F^i_v(u^\mathrm{ext})\right) \\
      91             :  * (n_i F^i_u)^* &= \frac{1}{2} n_i \left(
      92             :  * F^i_u(\partial_j F^j_v(u^\mathrm{int}) - S_v(u^\mathrm{int})) +
      93             :  * F^i_u(\partial_j F^j_v(u^\mathrm{ext}) - S_v(u^\mathrm{ext}))\right)
      94             :  * - \sigma n_i \left(
      95             :  * F^i_u(n_j F^j_v(u^\mathrm{int})) -
      96             :  * F^i_u(n_j F^j_v(u^\mathrm{ext}))\right)
      97             :  * \f}
      98             :  *
      99             :  * Note that \f$n_i\f$ denotes the face normal on the "interior" side of the
     100             :  * element under consideration. We assume \f$n^\mathrm{ext}_i=-n_i\f$ in the
     101             :  * implementation, i.e. face normals don't depend on the dynamic variables
     102             :  * (which may be discontinuous on element faces). This is the case for the
     103             :  * problems we are expecting to solve, because those will be on fixed background
     104             :  * metrics (e.g. a conformal metric for the XCTS system). Numerically, the face
     105             :  * normals on either side of a mortar may nonetheless be different because the
     106             :  * two faces adjacent to the mortar may resolve them at different resolutions.
     107             :  *
     108             :  * Also note that the numerical fluxes intentionally don't depend on the
     109             :  * auxiliary field values \f$v\f$. This property allows us to communicate data
     110             :  * for both the primal and auxiliary boundary corrections together, instead of
     111             :  * communicating them in two steps. If we were to resort to a two-step
     112             :  * communication we could replace the divergence in \f$(n_i F^i_u)^*\f$ with
     113             :  * \f$v\f$, which would result in a generalized "stabilized central flux" that
     114             :  * is slightly less sparse than the internal penalty flux (see e.g.
     115             :  * \cite HesthavenWarburton, section 7.2). We could also choose to ignore the
     116             :  * fluxes in the penalty term, but preliminary tests suggest that this may hurt
     117             :  * convergence.
     118             :  *
     119             :  * For a Poisson system (see `Poisson::FirstOrderSystem`) this numerical flux
     120             :  * reduces to the standard internal penalty flux (see e.g.
     121             :  * \cite HesthavenWarburton, section 7.2, or \cite Arnold2002):
     122             :  *
     123             :  * \f{align}
     124             :  * (n_i F^i_{v_j})^* &= n_j u^* = \frac{1}{2} n_j \left(
     125             :  * u^\mathrm{int} + u^\mathrm{ext}\right) \\
     126             :  * (n_i F^i_u)^* &= n_i v_i^* = \frac{1}{2} n_i \left(
     127             :  * \partial_i u^\mathrm{int} + \partial_i u^\mathrm{ext}\right)
     128             :  * - \sigma \left(u^\mathrm{int} - u^\mathrm{ext}\right)
     129             :  * \f}
     130             :  *
     131             :  * where a sum over repeated indices is assumed, since the equation is
     132             :  * formulated on a Euclidean geometry.
     133             :  *
     134             :  * The penalty factor \f$\sigma\f$ is responsible for removing zero eigenmodes
     135             :  * and impacts the conditioning of the linear operator to be solved. See
     136             :  * `elliptic::dg::penalty` for details. For the element size that goes into
     137             :  * computing the penalty we choose
     138             :  * \f$h=\frac{J_\mathrm{volume}}{J_\mathrm{face}}\f$, i.e. the ratio of Jacobi
     139             :  * determinants from logical to inertial coordinates in the element volume and
     140             :  * on the element face, both evaluated on the face (see \cite Vincent2019qpd).
     141             :  * Since both \f$N_\mathrm{points}\f$ and \f$h\f$ can be different on either
     142             :  * side of the element boundary we take the maximum of \f$N_\mathrm{points}\f$
     143             :  * and the pointwise minimum of \f$h\f$ across the element boundary as is done
     144             :  * in \cite Vincent2019qpd. Note that we use the number of points
     145             :  * \f$N_\mathrm{points}\f$ where \cite Vincent2019qpd uses the polynomial degree
     146             :  * \f$N_\mathrm{points} - 1\f$ because we found unstable configurations on
     147             :  * curved meshes when using the polynomial degree. Optimizing the penalty on
     148             :  * curved meshes is subject to further investigation.
     149             :  */
